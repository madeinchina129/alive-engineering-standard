# Project Glossary

## 通用术语

| 术语 | 英文 | 说明 |
|------|------|------|
| 架构决策记录 | ADR (Architecture Decision Record) | 记录架构决策的上下文、方案和理由 |
| 持续集成 | CI (Continuous Integration) | 频繁合并代码并自动构建测试 |
| 持续交付 | CD (Continuous Delivery) | 自动化部署流程 |
| 代码评审 | Code Review / PR Review | 对代码变更的同行审查 |
| 拉取请求 | PR (Pull Request) | 代码变更请求 |
| 技术债务 | Tech Debt | 为了短期效率而做出的妥协 |
| 验收标准 | Acceptance Criteria | 功能完成的条件定义 |

## 工程术语

| 术语 | 英文 | 说明 |
|------|------|------|
| 单体仓库 | Monorepo | 多个项目在同一个仓库中管理 |
| 微服务 | Microservices | 独立的服务架构风格 |
| 契约测试 | Contract Testing | 验证服务间接口一致性的测试 |
| 圈复杂度 | Cyclomatic Complexity | 代码逻辑复杂度的度量 |
| 关注点分离 | Separation of Concerns | 将不同功能模块分离的设计原则 |
| 单一职责 | Single Responsibility | 一个模块只负责一个功能 |

## 项目特定术语

| 术语 | 说明 |
|------|------|
| Alive Standard | 本工程标准规范体系 |
| Standard Document | 标准规范文档，位于 `docs/` 下 |
| Template | 可复用的文档/代码模板 |
| Checklist | 检查清单，用于质量门禁 |
| Context | 供 AI 或新成员理解的上下文文档 |
| Playbook | 操作手册，描述标准操作流程 |
| Generator | 脚手架生成器，用于初始化项目 |
