# Project Overview

## 项目定位

Alive Engineering Standard 是一个面向现代软件工程团队的**工程标准规范体系**。它不只是一个文档集合，而是一套可执行的、可维护的、可持续演进的工程方法论。

## 核心理念

| 原则 | 说明 |
|------|------|
| **文档驱动** | 先写文档，再写代码。文档是真实来源（Source of Truth） |
| **约定优于配置** | 明确约定规范，减少团队决策成本 |
| **渐进式采用** | 团队可以逐步采纳标准，无需一次性全盘接受 |
| **持续演进** | 标准本身通过 ADR 和版本管理持续迭代 |

## 项目范围

本规范涵盖：

- **项目管理规范** — 分支策略、Issue 管理、版本发布
- **产品设计规范** — PRD 模板、设计评审、体验标准
- **领域建模规范** — 实体定义、业务规则、领域事件
- **架构设计规范** — 分层架构、模块划分、接口契约
- **编码规范** — 语言/框架特定的编码标准
- **文档规范** — 文档结构、写作风格、模板
- **CI/CD 规范** — 流水线设计、自动化测试、部署策略
- **安全规范** — 安全评审、依赖管理、合规要求

## 目标受众

- 软件开发工程师
- 技术管理者（Tech Lead / EM）
- 产品经理
- DevOps / 平台工程师
- 新成员入职引导

## 成功标准

1. 新成员入职后 3 天内能独立完成一次规范提交
2. 团队 PR 评审效率提升 50%
3. 跨项目协作时减少 80% 的沟通成本
4. 标准覆盖率达到 90% 以上工程场景
