# Rust 测试 Checklist

## 单元测试
- [ ] 是否在 `#[cfg(test)] mod tests` 中？
- [ ] 是否测试了公共 API 和私有函数？
- [ ] 每个测试是否有明确的 assert？
- [ ] 测试是否使用 `Result<_, _>` 返回类型？

## 集成测试
- [ ] 集成测试是否放在 `tests/` 目录？
- [ ] 集成测试是否只测试公共 API？
- [ ] 是否避免集成测试依赖外部网络？
- [ ] 集成测试是否使用 `common/mod.rs` 共享辅助函数？

## 文档测试
- [ ] 所有公开 API 是否有文档测试？
- [ ] 文档测试是否包含 assert？
- [ ] 文档测试是否在 `cargo test` 中通过？

## 测试质量
- [ ] 是否避免 println 调试代码？
- [ ] 是否使用 `assert_eq!` / `assert_ne!` 而非手动比较？
- [ ] 测试是否有描述性名称 `test_功能_场景_期望结果`？
- [ ] 是否避免测试间共享可变状态？

## Mock 与测试工具
- [ ] 是否使用 mockall 或手动 mock trait？
- [ ] 异步测试是否使用 `#[tokio::test]`？
- [ ] 是否使用了 `#[should_panic]` 替代 Result 返回？

## 覆盖率
- [ ] 核心业务逻辑覆盖率 > 90%？
- [ ] 是否运行 `cargo test` 全部通过？
