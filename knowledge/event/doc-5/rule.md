# 消息中间件选型 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| EDA-MQ-001 | 消息中间件选型必须考虑：吞吐量、延迟、持久性、可用性、顺序保证 | P0 | 是 |
| EDA-MQ-002 | 核心业务消息必须通过选型评审，包含 POC 验证 | P0 | 是 |
| EDA-MQ-003 | 消息大小超过 1MB 必须使用对象存储引用方式 | P0 | 是 |
| EDA-MQ-004 | 消息中间件必须部署为集群模式，保障高可用 | P0 | 是 |

## 详细说明

### EDA-MQ-001（P0）
消息中间件选型必须考虑：吞吐量、延迟、持久性、可用性、顺序保证

### EDA-MQ-002（P0）
核心业务消息必须通过选型评审，包含 POC 验证

### EDA-MQ-003（P0）
消息大小超过 1MB 必须使用对象存储引用方式

### EDA-MQ-004（P0）
消息中间件必须部署为集群模式，保障高可用
