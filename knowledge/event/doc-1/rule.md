# 事件驱动架构原则 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| EDA-PR-001 | 核心业务实体状态变更必须发布领域事件 | P0 | 是 |
| EDA-PR-002 | 事件生产者和消费者之间必须通过消息中间件解耦 | P0 | 是 |
| EDA-PR-003 | 事件必须包含完整的业务上下文，消费者不应额外查询 | P0 | 是 |
| EDA-PR-004 | 事件消费必须实现幂等性（去重机制） | P0 | 是 |
| EDA-PR-005 | 事件处理失败必须有重试和死信队列机制 | P0 | 是 |

## 详细说明

### EDA-PR-001（P0）
核心业务实体状态变更必须发布领域事件

### EDA-PR-002（P0）
事件生产者和消费者之间必须通过消息中间件解耦

### EDA-PR-003（P0）
事件必须包含完整的业务上下文，消费者不应额外查询

### EDA-PR-004（P0）
事件消费必须实现幂等性（去重机制）

### EDA-PR-005（P0）
事件处理失败必须有重试和死信队列机制
