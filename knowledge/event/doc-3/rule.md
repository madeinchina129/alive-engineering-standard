# 事件溯源模式 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| EDA-ES-001 | 事件存储必须是仅追加（Append-only），事件不可修改或删除 | P0 | 是 |
| EDA-ES-002 | 重大数据变更必须通过事件溯源记录 | P1 | 推荐 |
| EDA-ES-003 | 事件存储必须支持按聚合 ID 和事件序列号顺序读取 | P0 | 是 |
| EDA-ES-004 | 每 N 个事件后创建一次快照（建议 N=100） | P1 | 推荐 |
| EDA-ES-005 | 快照存储需压缩以减少存储成本 | P2 | 推荐 |

## 详细说明

### EDA-ES-001（P0）
事件存储必须是仅追加（Append-only），事件不可修改或删除

### EDA-ES-002（P1）
重大数据变更必须通过事件溯源记录

### EDA-ES-003（P0）
事件存储必须支持按聚合 ID 和事件序列号顺序读取

### EDA-ES-004（P1）
每 N 个事件后创建一次快照（建议 N=100）

### EDA-ES-005（P2）
快照存储需压缩以减少存储成本
