# AI Agent 设计规范 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| AI-AGT-001 | Agent 必须使用 ReAct（推理+行动）循环模式，而非直接输出 | P0 | 是 |
| AI-AGT-002 | Agent 的工具调用必须有参数校验和异常处理 | P0 | 是 |
| AI-AGT-003 | Agent 必须有最大步数限制（默认 10 步），防止无限循环 | P0 | 是 |
| AI-AGT-004 | Agent 涉及用户数据时必须有明确的授权确认流程 | P0 | 是 |
| AI-AGT-005 | Agent 的所有操作日志必须持久化存储用于审计 | P0 | 是 |
| AI-AGT-006 | 高危操作（删除/修改/支付）必须经用户二次确认 | P0 | 是 |

## 详细说明

### AI-AGT-001（P0）
Agent 必须使用 ReAct（推理+行动）循环模式，而非直接输出

### AI-AGT-002（P0）
Agent 的工具调用必须有参数校验和异常处理

### AI-AGT-003（P0）
Agent 必须有最大步数限制（默认 10 步），防止无限循环

### AI-AGT-004（P0）
Agent 涉及用户数据时必须有明确的授权确认流程

### AI-AGT-005（P0）
Agent 的所有操作日志必须持久化存储用于审计

### AI-AGT-006（P0）
高危操作（删除/修改/支付）必须经用户二次确认
