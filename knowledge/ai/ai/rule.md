# AI 安全规范 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| AI-SAFE-001 | 所有用户输入必须经过注入检测和敏感词过滤 | P0 | 是 |
| AI-SAFE-002 | 模型输出必须经过内容安全审核（暴力/色情/仇恨言论等） | P0 | 是 |
| AI-SAFE-003 | AI 系统不能执行未经用户授权的敏感操作 | P0 | 是 |
| AI-SAFE-004 | 训练数据必须过滤 PII（个人身份信息） | P0 | 是 |
| AI-SAFE-005 | 模型权重文件必须有访问控制和版本签名 | P0 | 是 |
| AI-SAFE-006 | AI 系统的日志必须保留至少 90 天用于安全审计 | P0 | 是 |

## 详细说明

### AI-SAFE-001（P0）
所有用户输入必须经过注入检测和敏感词过滤

### AI-SAFE-002（P0）
模型输出必须经过内容安全审核（暴力/色情/仇恨言论等）

### AI-SAFE-003（P0）
AI 系统不能执行未经用户授权的敏感操作

### AI-SAFE-004（P0）
训练数据必须过滤 PII（个人身份信息）

### AI-SAFE-005（P0）
模型权重文件必须有访问控制和版本签名

### AI-SAFE-006（P0）
AI 系统的日志必须保留至少 90 天用于安全审计
