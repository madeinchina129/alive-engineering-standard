# RAG 系统设计 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| AI-RAG-001 | 文档分块策略必须基于文档类型定制（Markdown 按标题、代码按函数、PDF 按段落） | P0 | 是 |
| AI-RAG-002 | 检索必须包含：BM25 关键词检索 + 向量语义检索 + Cross-Encoder 重排序 | P0 | 是 |
| AI-RAG-003 | 向量数据库必须定期更新索引，删除过期文档 | P0 | 是 |
| AI-RAG-004 | Top-K 检索数量控制在 3-8 个块，兼顾相关性和 Token 消耗 | P0 | 是 |
| AI-RAG-005 | 生成结果必须标注引用来源（文档 ID + 原文片段） | P0 | 是 |
| AI-RAG-006 | RAG Pipeline 必须有完整的可观测性（检索质量、生成质量、延迟） | P1 | 推荐 |

## 详细说明

### AI-RAG-001（P0）
文档分块策略必须基于文档类型定制（Markdown 按标题、代码按函数、PDF 按段落）

### AI-RAG-002（P0）
检索必须包含：BM25 关键词检索 + 向量语义检索 + Cross-Encoder 重排序

### AI-RAG-003（P0）
向量数据库必须定期更新索引，删除过期文档

### AI-RAG-004（P0）
Top-K 检索数量控制在 3-8 个块，兼顾相关性和 Token 消耗

### AI-RAG-005（P0）
生成结果必须标注引用来源（文档 ID + 原文片段）

### AI-RAG-006（P1）
RAG Pipeline 必须有完整的可观测性（检索质量、生成质量、延迟）
