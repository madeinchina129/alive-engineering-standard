# 模型服务规范 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| AI-SRV-001 | 模型服务必须支持健康检查和预热（Warm-up） | P0 | 是 |
| AI-SRV-002 | 推理请求必须设置超时（在线 5s / 离线 60s） | P0 | 是 |
| AI-SRV-003 | 模型推理使用 GPU 必须启用动态批处理（Dynamic Batching） | P0 | 是 |
| AI-SRV-004 | 生产环境部署至少 2 个模型副本保障高可用 | P0 | 是 |
| AI-SRV-005 | 模型版本切换使用蓝绿部署或金丝雀发布 | P0 | 是 |
| AI-SRV-006 | 推理结果必须缓存（相同输入命中缓存减少计算） | P1 | 推荐 |

## 详细说明

### AI-SRV-001（P0）
模型服务必须支持健康检查和预热（Warm-up）

### AI-SRV-002（P0）
推理请求必须设置超时（在线 5s / 离线 60s）

### AI-SRV-003（P0）
模型推理使用 GPU 必须启用动态批处理（Dynamic Batching）

### AI-SRV-004（P0）
生产环境部署至少 2 个模型副本保障高可用

### AI-SRV-005（P0）
模型版本切换使用蓝绿部署或金丝雀发布

### AI-SRV-006（P1）
推理结果必须缓存（相同输入命中缓存减少计算）
