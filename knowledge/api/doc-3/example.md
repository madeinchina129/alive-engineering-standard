```markdown
# API 版本管理指南

## 版本号规则
- v1.0: 初始版本
- v1.1: 新增字段（非破坏性）
- v1.2: 新增端点（非破坏性）
- v2.0: 破坏性变更

## 版本生命周期
| 阶段 | 状态 | 说明 |
|------|------|------|
| Active | ✅ 当前版本 | 完全支持，新功能开发 |
| Deprecated | ⚠️ 即将废弃 | 只修 Bug，不开发新功能 |
| Sunset | 🚫 已下线 | 不再提供服务 |

## 当前活跃版本
| 版本 | 状态 | 废弃日期 | 下线日期 |
|------|------|----------|----------|
| v1 | Active | - | - |
| v2 | Deprecated | 2024-01-15 | 2024-07-15 |

## 废弃通知模板
```
[NOTICE] API v2 废弃通知

亲爱的开发者,

API v2 版本将于 2024-07-15 正式下线。
请在 2024-07-15 前完成到 v3 版本的迁移。

变更摘要：
- POST /api/v2/orders → POST /api/v3/orders (响应体新增 fields 参数)
- 新增: GET /api/v3/orders/export 批量导出

迁移指南：https://docs.example.com/api/v3-migration

如有问题请联系 API 团队。
```

## 版本兼容性检查清单
- [ ] 新增字段：可选（非必填），不影响现有客户端
- [ ] 字段废弃：保留字段但文档标记 @deprecated
- [ ] 响应结构调整：保持原有结构不变，新增字段放在末尾
- [ ] 默认行为：不改变现有 API 的默认行为
```
