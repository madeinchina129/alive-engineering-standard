# API 版本管理规范 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| API-VER-001 | API 版本号放在 URL 路径中（如 /api/v1/users） | P0 | 是 |
| API-VER-002 | Major 版本变更必须提前 6 个月发出废弃通知 | P0 | 是 |
| API-VER-003 | 非破坏性变更（新增字段、新增端点）不需要升级版本号 | P0 | 是 |
| API-VER-004 | 废弃的 API 版本至少维护 6 个月后才可下线 | P0 | 是 |
| API-VER-005 | 每个 API 版本必须有对应的 Changelog 文档 | P0 | 是 |
| API-VER-006 | 客户端需要在请求头中声明其所使用的 API 版本 | P1 | 推荐 |

## 详细说明

### API-VER-001（P0）
API 版本号放在 URL 路径中（如 /api/v1/users）

### API-VER-002（P0）
Major 版本变更必须提前 6 个月发出废弃通知

### API-VER-003（P0）
非破坏性变更（新增字段、新增端点）不需要升级版本号

### API-VER-004（P0）
废弃的 API 版本至少维护 6 个月后才可下线

### API-VER-005（P0）
每个 API 版本必须有对应的 Changelog 文档

### API-VER-006（P1）
客户端需要在请求头中声明其所使用的 API 版本
