# 性能优化模式 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| PERF-OPT-001 | 性能优化前必须通过 Profiling 定位瓶颈，禁止凭猜测优化 | P0 | 是 |
| PERF-OPT-002 | 每次优化只修改一处，优化后必须重新测量对比 | P0 | 是 |
| PERF-OPT-003 | 缓存策略需考虑：缓存命中率、过期策略、缓存穿透/击穿/雪崩防护 | P0 | 是 |
| PERF-OPT-004 | N+1 查询问题必须通过批量查询或懒加载优化 | P0 | 是 |
| PERF-OPT-005 | 数据库慢查询（> 1s）必须优化，长期慢查询必须有优化计划 | P0 | 是 |

## 详细说明

### PERF-OPT-001（P0）
性能优化前必须通过 Profiling 定位瓶颈，禁止凭猜测优化

### PERF-OPT-002（P0）
每次优化只修改一处，优化后必须重新测量对比

### PERF-OPT-003（P0）
缓存策略需考虑：缓存命中率、过期策略、缓存穿透/击穿/雪崩防护

### PERF-OPT-004（P0）
N+1 查询问题必须通过批量查询或懒加载优化

### PERF-OPT-005（P0）
数据库慢查询（> 1s）必须优化，长期慢查询必须有优化计划
