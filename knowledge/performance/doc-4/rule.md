# 容量规划 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| PERF-CAP-001 | 容量规划每季度执行一次，覆盖核心业务系统 | P0 | 是 |
| PERF-CAP-002 | 容量规划必须基于：当前使用率、历史增长趋势、业务增长预期 | P0 | 是 |
| PERF-CAP-003 | 资源使用率超过 70% 时需启动扩容评估 | P0 | 是 |
| PERF-CAP-004 | 关键业务系统必须配置自动扩容策略 | P1 | 推荐 |
| PERF-CAP-005 | 每年至少进行一次全系统压力测试验证容量上限 | P1 | 推荐 |

## 详细说明

### PERF-CAP-001（P0）
容量规划每季度执行一次，覆盖核心业务系统

### PERF-CAP-002（P0）
容量规划必须基于：当前使用率、历史增长趋势、业务增长预期

### PERF-CAP-003（P0）
资源使用率超过 70% 时需启动扩容评估

### PERF-CAP-004（P1）
关键业务系统必须配置自动扩容策略

### PERF-CAP-005（P1）
每年至少进行一次全系统压力测试验证容量上限
