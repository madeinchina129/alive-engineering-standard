# 设计系统管理 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| UI-DSM-001 | 设计系统使用 Semver 语义化版本（major.minor.patch） | P0 | 是 |
| UI-DSM-002 | 每个组件必须有使用统计，使用率低于 10% 的组件标记为废弃 | P0 | 是 |
| UI-DSM-003 | 组件发布需经过：设计评审 → 开发实现 → 视觉审查 → 文档更新 → 发布 | P0 | 是 |
| UI-DSM-004 | 设计系统和代码库保持同步，Figma 组件库和 npm 包版本一致 | P1 | 是 |
| UI-DSM-005 | 每月进行设计系统健康度检查，输出治理报告 | P2 | 推荐 |

## 详细说明

### UI-DSM-001（P0）
设计系统使用 Semver 语义化版本（major.minor.patch）

### UI-DSM-002（P0）
每个组件必须有使用统计，使用率低于 10% 的组件标记为废弃

### UI-DSM-003（P0）
组件发布需经过：设计评审 → 开发实现 → 视觉审查 → 文档更新 → 发布

### UI-DSM-004（P1）
设计系统和代码库保持同步，Figma 组件库和 npm 包版本一致

### UI-DSM-005（P2）
每月进行设计系统健康度检查，输出治理报告
