# 能力视图管理 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| CAP-VW-001 | L1 能力域不超过 8 个，每个 L1 下 L2 不超过 6 个 | P0 | 是 |
| CAP-VW-002 | 能力命名采用 动宾结构（如「订单管理」「风险控制」） | P0 | 是 |
| CAP-VW-003 | 每个能力必须明确：能力描述、所属域、依赖关系 | P0 | 是 |
| CAP-VW-004 | 能力视图至少每年评审更新一次，与业务规划对齐 | P1 | 推荐 |

## 详细说明

### CAP-VW-001（P0）
L1 能力域不超过 8 个，每个 L1 下 L2 不超过 6 个

### CAP-VW-002（P0）
能力命名采用 动宾结构（如「订单管理」「风险控制」）

### CAP-VW-003（P0）
每个能力必须明确：能力描述、所属域、依赖关系

### CAP-VW-004（P1）
能力视图至少每年评审更新一次，与业务规划对齐
