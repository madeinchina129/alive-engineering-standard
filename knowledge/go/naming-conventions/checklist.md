# Go 命名规范 Checklist

## 包命名
- [ ] 包名是否小写、单数？
- [ ] 包名是否描述功能而非内容？
- [ ] 包名是否避免 `util`/`common`/`helpers`？
- [ ] 包名是否和目录名一致？

## 类型命名
- [ ] 公开类型是否 PascalCase？
- [ ] 私有类型是否 camelCase？
- [ ] 单方法接口是否 -er 结尾？
- [ ] 多方法接口是否有完整清晰名称？

## 函数/方法命名
- [ ] 公开函数是否 PascalCase？
- [ ] 私有函数是否 camelCase？
- [ ] 函数名是否动词或动词+名词？
- [ ] 是否避免 Get 冗余（包名+函数名重复）？
- [ ] 接收者是否 1-3 字母缩写？
- [ ] 接收者是否避免 `self`/`this`？

## 变量命名
- [ ] 局部变量是否 camelCase？
- [ ] 变量名是否随作用域变小而变短？
- [ ] 布尔变量是否 Is/Has/Can 开头？
- [ ] 是否避免下划线命名（测试文件除外）？

## 常量命名
- [ ] 常量是否 PascalCase？
- [ ] 是否避免全大写（`MAX_RETRIES`）？
- [ ] 是否避免下划线（`default_port`）？

## 通用
- [ ] `golint` 是否无 warning？
- [ ] `go vet` 是否通过？
- [ ] 代码是否通过 `staticcheck`？
