# Go 错误处理 Checklist

## 错误检查
- [ ] 每个返回 error 的调用是否被检查？
- [ ] 是否避免 `_` 忽略 error？
- [ ] 是否避免 panic 处理可恢复错误？
- [ ] 是否只处理一次（不 log + return 两次）？

## 错误包装
- [ ] 是否使用 `%w` 而非 `%v` 包装错误？
- [ ] 错误消息是否包含足够的上下文信息？
- [ ] 多层调用是否逐层包装形成错误链？

## 错误判断
- [ ] 是否使用 `errors.Is` 判断 sentinel error？
- [ ] 是否使用 `errors.As` 提取自定义错误？
- [ ] 是否避免直接 `==` 比较可能被包装的错误？

## 自定义错误
- [ ] 自定义错误是否实现 `Error()` 方法？
- [ ] 需要包装时是否实现 `Unwrap()`？
- [ ] 是否在合适的地方定义了 sentinel error？

## 日志与错误分离
- [ ] 业务层是否只返回不 log？
- [ ] HTTP/gRPC 层是否统一 log？
- [ ] 是否保持了用户友好的错误消息？
