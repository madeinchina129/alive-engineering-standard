# 端到端测试规范 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| TST-E2E-001 | E2E 测试覆盖核心业务路径（注册→登录→下单→支付→完成） | P0 | 是 |
| TST-E2E-002 | 元素定位使用 data-testid 属性，不使用 CSS class 或 XPath | P0 | 是 |
| TST-E2E-003 | E2E 测试运行在独立测试环境，不影响生产数据 | P0 | 是 |
| TST-E2E-004 | 每次 PR 合并前自动运行核心 E2E 用例（< 15 分钟） | P0 | 是 |
| TST-E2E-005 | 全量 E2E 回归测试每天定时运行一次 | P1 | 推荐 |
| TST-E2E-006 | E2E 测试失败自动截图和录制视频 | P1 | 是 |

## 详细说明

### TST-E2E-001（P0）
E2E 测试覆盖核心业务路径（注册→登录→下单→支付→完成）

### TST-E2E-002（P0）
元素定位使用 data-testid 属性，不使用 CSS class 或 XPath

### TST-E2E-003（P0）
E2E 测试运行在独立测试环境，不影响生产数据

### TST-E2E-004（P0）
每次 PR 合并前自动运行核心 E2E 用例（< 15 分钟）

### TST-E2E-005（P1）
全量 E2E 回归测试每天定时运行一次

### TST-E2E-006（P1）
E2E 测试失败自动截图和录制视频
