# 单元测试规范 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| TST-UT-001 | 单元测试必须与源代码一同提交，不允许事后补测试 | P0 | 是 |
| TST-UT-002 | 核心逻辑代码单元测试覆盖率 ≥ 90%，整体 ≥ 80% | P0 | 是 |
| TST-UT-003 | 测试方法命名使用 test[场景]_[期望结果] 格式 | P0 | 是 |
| TST-UT-004 | 每个测试只使用一个 Mock 对象，避免过度 Mock | P0 | 是 |
| TST-UT-005 | 测试数据使用 Builder 模式构造，不使用真实数据 | P1 | 推荐 |
| TST-UT-006 | 私有方法不直接测试，通过公有接口测覆盖 | P1 | 是 |

## 详细说明

### TST-UT-001（P0）
单元测试必须与源代码一同提交，不允许事后补测试

### TST-UT-002（P0）
核心逻辑代码单元测试覆盖率 ≥ 90%，整体 ≥ 80%

### TST-UT-003（P0）
测试方法命名使用 test[场景]_[期望结果] 格式

### TST-UT-004（P0）
每个测试只使用一个 Mock 对象，避免过度 Mock

### TST-UT-005（P1）
测试数据使用 Builder 模式构造，不使用真实数据

### TST-UT-006（P1）
私有方法不直接测试，通过公有接口测覆盖
