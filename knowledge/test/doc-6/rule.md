# 测试自动化规范 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| TST-AUTO-001 | CI 流水线中必须包含自动化测试阶段 | P0 | 是 |
| TST-AUTO-002 | 单元测试在每次代码提交时自动运行 | P0 | 是 |
| TST-AUTO-003 | 集成测试在每个 PR 创建时自动运行 | P0 | 是 |
| TST-AUTO-004 | E2E 测试在 PR 合并前运行（可选：只在包含前端变更时运行） | P0 | 是 |
| TST-AUTO-005 | 测试报告必须量化和追踪：通过率、失败趋势、运行时间 | P1 | 是 |
| TST-AUTO-006 | Flaky 测试必须标记并修复，不超过总测试数的 1% | P1 | 是 |

## 详细说明

### TST-AUTO-001（P0）
CI 流水线中必须包含自动化测试阶段

### TST-AUTO-002（P0）
单元测试在每次代码提交时自动运行

### TST-AUTO-003（P0）
集成测试在每个 PR 创建时自动运行

### TST-AUTO-004（P0）
E2E 测试在 PR 合并前运行（可选：只在包含前端变更时运行）

### TST-AUTO-005（P1）
测试报告必须量化和追踪：通过率、失败趋势、运行时间

### TST-AUTO-006（P1）
Flaky 测试必须标记并修复，不超过总测试数的 1%
