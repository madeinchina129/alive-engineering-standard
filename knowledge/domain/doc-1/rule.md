# 限界上下文设计 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| DDD-BC-001 | 限界上下文必须基于业务能力和领域专家语言划分 | P0 | 是 |
| DDD-BC-002 | 每个上下文必须有明确的职责说明和边界定义 | P0 | 是 |
| DDD-BC-003 | 上下文关系必须使用上下文映射图标注（合作关系/上下游/共享内核等） | P0 | 是 |
| DDD-BC-004 | 上下文间的通信最多允许两种方式：事件驱动或 API 调用 | P1 | 推荐 |

## 详细说明

### DDD-BC-001（P0）
限界上下文必须基于业务能力和领域专家语言划分

### DDD-BC-002（P0）
每个上下文必须有明确的职责说明和边界定义

### DDD-BC-003（P0）
上下文关系必须使用上下文映射图标注（合作关系/上下游/共享内核等）

### DDD-BC-004（P1）
上下文间的通信最多允许两种方式：事件驱动或 API 调用
