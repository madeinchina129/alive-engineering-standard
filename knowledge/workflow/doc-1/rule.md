# 状态机模式 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| WF-SM-001 | 所有状态机必须显式定义：状态集、事件集、转换规则、初始状态、终态 | P0 | 是 |
| WF-SM-002 | 非法状态转换必须被框架拒绝并抛出明确异常 | P0 | 是 |
| WF-SM-003 | 状态转换前后可以有 Action（前处理/后处理） | P0 | 是 |
| WF-SM-004 | 状态机的定义必须集中管理，禁止分散在多个文件中 | P0 | 是 |
| WF-SM-005 | 关键业务对象的状态变更必须发布领域事件 | P1 | 推荐 |

## 详细说明

### WF-SM-001（P0）
所有状态机必须显式定义：状态集、事件集、转换规则、初始状态、终态

### WF-SM-002（P0）
非法状态转换必须被框架拒绝并抛出明确异常

### WF-SM-003（P0）
状态转换前后可以有 Action（前处理/后处理）

### WF-SM-004（P0）
状态机的定义必须集中管理，禁止分散在多个文件中

### WF-SM-005（P1）
关键业务对象的状态变更必须发布领域事件
