# 工作流引擎 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| WF-ENG-001 | 工作流定义必须使用 BPMN 2.0 标准建模 | P0 | 是 |
| WF-ENG-002 | 工作流引擎必须支持：用户任务、服务任务、网关（排他/并行/包容）、子流程 | P0 | 是 |
| WF-ENG-003 | 流程实例状态必须包括：运行中、挂起、已完成、已终止 | P0 | 是 |
| WF-ENG-004 | 超时未处理的用户任务必须发送催办通知 | P0 | 是 |
| WF-ENG-005 | 工作流引擎必须提供 REST API 进行流程操作 | P1 | 推荐 |

## 详细说明

### WF-ENG-001（P0）
工作流定义必须使用 BPMN 2.0 标准建模

### WF-ENG-002（P0）
工作流引擎必须支持：用户任务、服务任务、网关（排他/并行/包容）、子流程

### WF-ENG-003（P0）
流程实例状态必须包括：运行中、挂起、已完成、已终止

### WF-ENG-004（P0）
超时未处理的用户任务必须发送催办通知

### WF-ENG-005（P1）
工作流引擎必须提供 REST API 进行流程操作
