# 监控告警标准 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| OPS-MON-001 | 每个服务必须监控：CPU、内存、GC、请求数、错误率、响应时间 | P0 | 是 |
| OPS-MON-002 | 告警分为 P0（紧急）/P1（严重）/P2（警告）/P3（通知）四级 | P0 | 是 |
| OPS-MON-003 | P0 告警必须 5 分钟内通知到值班人员，P1 告警 15 分钟内 | P0 | 是 |
| OPS-MON-004 | 告警必须有明确的解决步骤文档（Runbook） | P0 | 是 |
| OPS-MON-005 | 每周统计告警噪音率（误报/无需行动的告警），目标 < 10% | P1 | 推荐 |

## 详细说明

### OPS-MON-001（P0）
每个服务必须监控：CPU、内存、GC、请求数、错误率、响应时间

### OPS-MON-002（P0）
告警分为 P0（紧急）/P1（严重）/P2（警告）/P3（通知）四级

### OPS-MON-003（P0）
P0 告警必须 5 分钟内通知到值班人员，P1 告警 15 分钟内

### OPS-MON-004（P0）
告警必须有明确的解决步骤文档（Runbook）

### OPS-MON-005（P1）
每周统计告警噪音率（误报/无需行动的告警），目标 < 10%
