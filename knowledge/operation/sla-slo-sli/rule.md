# SLA/SLO/SLI 体系 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| OPS-SLO-001 | 每个外部面向的服务必须定义 SLI、SLO 和 SLA | P0 | 是 |
| OPS-SLO-002 | SLI 包括：可用性、延迟、吞吐量、错误率、饱和度 | P0 | 是 |
| OPS-SLO-003 | SLO 目标建议值：可用性 99.9%+、P99 延迟 < 1s、错误率 < 0.1% | P0 | 是 |
| OPS-SLO-004 | 错误预算月度消耗 100% 时应触发变更冻结 | P0 | 是 |
| OPS-SLO-005 | SLO 达标情况必须每月审查并报告给业务方 | P1 | 推荐 |

## 详细说明

### OPS-SLO-001（P0）
每个外部面向的服务必须定义 SLI、SLO 和 SLA

### OPS-SLO-002（P0）
SLI 包括：可用性、延迟、吞吐量、错误率、饱和度

### OPS-SLO-003（P0）
SLO 目标建议值：可用性 99.9%+、P99 延迟 < 1s、错误率 < 0.1%

### OPS-SLO-004（P0）
错误预算月度消耗 100% 时应触发变更冻结

### OPS-SLO-005（P1）
SLO 达标情况必须每月审查并报告给业务方
