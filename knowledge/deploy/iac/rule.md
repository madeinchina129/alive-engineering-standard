# 监控与告警规范 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| DEP-MON-001 | 所有服务必须暴露 /health（存活）和 /metrics（Prometheus）端点 | P0 | 是 |
| DEP-MON-002 | 核心 RED 指标必须配置告警：错误率 > 1%、P95 延迟 > 1s、QPS 突降 > 50% | P0 | 是 |
| DEP-MON-003 | P0 告警通过电话/短信通知值班人员，P1 通过即时通信通知 | P0 | 是 |
| DEP-MON-004 | 所有系统日志使用结构化格式（JSON），统一采集到中心化日志平台 | P0 | 是 |
| DEP-MON-005 | 告警规则必须配置冷却时间（至少 5 分钟），防止重复告警 | P0 | 是 |
| DEP-MON-006 | 每月进行告警规则有效性审查，清除无效/噪音告警 | P1 | 推荐 |

## 详细说明

### DEP-MON-001（P0）
所有服务必须暴露 /health（存活）和 /metrics（Prometheus）端点

### DEP-MON-002（P0）
核心 RED 指标必须配置告警：错误率 > 1%、P95 延迟 > 1s、QPS 突降 > 50%

### DEP-MON-003（P0）
P0 告警通过电话/短信通知值班人员，P1 通过即时通信通知

### DEP-MON-004（P0）
所有系统日志使用结构化格式（JSON），统一采集到中心化日志平台

### DEP-MON-005（P0）
告警规则必须配置冷却时间（至少 5 分钟），防止重复告警

### DEP-MON-006（P1）
每月进行告警规则有效性审查，清除无效/噪音告警
