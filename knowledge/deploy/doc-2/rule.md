# CI/CD 流水线规范 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| DEP-CI-001 | 每次代码提交必须触发自动化流水线（构建 + 测试 + 静态分析） | P0 | 是 |
| DEP-CI-002 | CI 流水线必须在 10 分钟内完成（单元测试 + 代码检查） | P0 | 是 |
| DEP-CI-003 | 流水线失败必须阻断合并，不允许跳过质量门禁 | P0 | 是 |
| DEP-CI-004 | 每个构建产物必须有唯一版本号（Git SHA + Build Number） | P0 | 是 |
| DEP-CI-005 | 部署流水线必须有审批机制（环境越靠近生产，审批越严格） | P0 | 是 |
| DEP-CI-006 | 流水线配置（Jenkinsfile/GitHub Actions YAML）与代码一同版本管理 | P0 | 是 |

## 详细说明

### DEP-CI-001（P0）
每次代码提交必须触发自动化流水线（构建 + 测试 + 静态分析）

### DEP-CI-002（P0）
CI 流水线必须在 10 分钟内完成（单元测试 + 代码检查）

### DEP-CI-003（P0）
流水线失败必须阻断合并，不允许跳过质量门禁

### DEP-CI-004（P0）
每个构建产物必须有唯一版本号（Git SHA + Build Number）

### DEP-CI-005（P0）
部署流水线必须有审批机制（环境越靠近生产，审批越严格）

### DEP-CI-006（P0）
流水线配置（Jenkinsfile/GitHub Actions YAML）与代码一同版本管理
