# 关系型数据库设计规范 — FAQ

## Q1: 自增 ID 还是 UUID？
自增 ID：性能好、索引小、适合内部系统。UUID：适合分布式系统、避免暴露业务量级。折衷方案：使用雪花算法（Snowflake ID）生成分布式唯一 ID。

## Q2: 反范式化什么时候用？
当查询性能成为瓶颈，且数据一致性可以通过应用层保证时。典型场景：冗余展示字段（如订单表中的用户名称）、预计算聚合（如文章表中的评论数）。

## Q3: 删除数据是物理删除还是逻辑删除？
推荐逻辑删除（is_deleted 字段），保留数据可追溯。对于需要物理删除的场景（如 GDPR 合规），使用定时清理任务批量处理。
