# 数据建模方法 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| DB-DM-001 | 所有项目必须有数据字典，定义每个字段的含义、类型、取值范围 | P0 | 是 |
| DB-DM-002 | 核心业务实体必须使用 ER 图进行建模 | P0 | 是 |
| DB-DM-003 | 枚举字段的取值范围必须文档化，代码中定义为常量 | P0 | 是 |
| DB-DM-004 | 金额字段使用 DECIMAL(12,2) 或 BIGINT（分单位），不使用 FLOAT/DOUBLE | P0 | 是 |
| DB-DM-005 | 时间字段统一使用 UTC 存储，展示时按用户时区转换 | P0 | 是 |
| DB-DM-006 | 敏感字段（密码/手机/身份证）必须加密存储 | P0 | 是 |

## 详细说明

### DB-DM-001（P0）
所有项目必须有数据字典，定义每个字段的含义、类型、取值范围

### DB-DM-002（P0）
核心业务实体必须使用 ER 图进行建模

### DB-DM-003（P0）
枚举字段的取值范围必须文档化，代码中定义为常量

### DB-DM-004（P0）
金额字段使用 DECIMAL(12,2) 或 BIGINT（分单位），不使用 FLOAT/DOUBLE

### DB-DM-005（P0）
时间字段统一使用 UTC 存储，展示时按用户时区转换

### DB-DM-006（P0）
敏感字段（密码/手机/身份证）必须加密存储
