# 认证与授权规范 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| SEC-AUTH-001 | 用户密码必须使用 bcrypt/Argon2 等慢哈希算法存储 | P0 | 是 |
| SEC-AUTH-002 | API 认证必须使用 JWT/OAuth 2.0 等标准协议 | P0 | 是 |
| SEC-AUTH-003 | 所有 API 请求必须经过身份验证（除公开接口） | P0 | 是 |
| SEC-AUTH-004 | 账户登录失败 5 次后必须临时锁定（至少 15 分钟） | P0 | 是 |
| SEC-AUTH-005 | 未操作 30 分钟后会话必须过期 | P0 | 是 |
| SEC-AUTH-006 | 权限模型采用 RBAC（基于角色的访问控制） | P1 | 推荐 |

## 详细说明

### SEC-AUTH-001（P0）
用户密码必须使用 bcrypt/Argon2 等慢哈希算法存储

### SEC-AUTH-002（P0）
API 认证必须使用 JWT/OAuth 2.0 等标准协议

### SEC-AUTH-003（P0）
所有 API 请求必须经过身份验证（除公开接口）

### SEC-AUTH-004（P0）
账户登录失败 5 次后必须临时锁定（至少 15 分钟）

### SEC-AUTH-005（P0）
未操作 30 分钟后会话必须过期

### SEC-AUTH-006（P1）
权限模型采用 RBAC（基于角色的访问控制）
