# 数据加密方案 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| SEC-CRYPTO-001 | 传输中的数据必须使用 TLS 1.2+ 加密 | P0 | 是 |
| SEC-CRYPTO-002 | 存储的敏感数据（PII/金融数据）必须使用 AES-256 加密 | P0 | 是 |
| SEC-CRYPTO-003 | 密钥必须使用 KMS（密钥管理服务）管理，禁止硬编码在代码中 | P0 | 是 |
| SEC-CRYPTO-004 | 加密密钥必须定期轮换（建议 90 天轮换一次） | P0 | 是 |
| SEC-CRYPTO-005 | 数据库敏感字段必须使用列级加密 | P1 | 推荐 |

## 详细说明

### SEC-CRYPTO-001（P0）
传输中的数据必须使用 TLS 1.2+ 加密

### SEC-CRYPTO-002（P0）
存储的敏感数据（PII/金融数据）必须使用 AES-256 加密

### SEC-CRYPTO-003（P0）
密钥必须使用 KMS（密钥管理服务）管理，禁止硬编码在代码中

### SEC-CRYPTO-004（P0）
加密密钥必须定期轮换（建议 90 天轮换一次）

### SEC-CRYPTO-005（P1）
数据库敏感字段必须使用列级加密
