# 安全审计规范 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| SEC-AUD-001 | 安全审计至少每季度执行一次 | P0 | 是 |
| SEC-AUD-002 | 审计范围必须覆盖：基础设施安全、应用安全、数据安全、权限安全 | P0 | 是 |
| SEC-AUD-003 | 发现的安全漏洞必须在修复时限内完成修复 | P0 | 是 |
| SEC-AUD-004 | 高危漏洞修复时限为 7 天，中危 30 天，低危 90 天 | P0 | 是 |
| SEC-AUD-005 | 审计报告必须提交安全委员会并归档 | P0 | 是 |

## 详细说明

### SEC-AUD-001（P0）
安全审计至少每季度执行一次

### SEC-AUD-002（P0）
审计范围必须覆盖：基础设施安全、应用安全、数据安全、权限安全

### SEC-AUD-003（P0）
发现的安全漏洞必须在修复时限内完成修复

### SEC-AUD-004（P0）
高危漏洞修复时限为 7 天，中危 30 天，低危 90 天

### SEC-AUD-005（P0）
审计报告必须提交安全委员会并归档
