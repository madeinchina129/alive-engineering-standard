# 密钥管理规范 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| SEC-KM-001 | 所有密钥必须通过 KMS（云 KMS/HashiCorp Vault）管理 | P0 | 是 |
| SEC-KM-002 | 密钥禁止以明文形式存储在代码库、配置文件或环境变量中 | P0 | 是 |
| SEC-KM-003 | 生产环境密钥与开发/测试环境密钥必须完全隔离 | P0 | 是 |
| SEC-KM-004 | 密钥访问必须有审计日志记录：谁、何时、为何、访问了哪个密钥 | P0 | 是 |
| SEC-KM-005 | 密钥轮换周期：加密密钥 90 天、签名密钥 180 天、CA 密钥 365 天 | P0 | 是 |

## 详细说明

### SEC-KM-001（P0）
所有密钥必须通过 KMS（云 KMS/HashiCorp Vault）管理

### SEC-KM-002（P0）
密钥禁止以明文形式存储在代码库、配置文件或环境变量中

### SEC-KM-003（P0）
生产环境密钥与开发/测试环境密钥必须完全隔离

### SEC-KM-004（P0）
密钥访问必须有审计日志记录：谁、何时、为何、访问了哪个密钥

### SEC-KM-005（P0）
密钥轮换周期：加密密钥 90 天、签名密钥 180 天、CA 密钥 365 天
