# React 组件 Code Review Checklist

## 组件结构
- [ ] 组件是否单一职责？（不超过 100 行）
- [ ] 是否一个组件一个目录？
- [ ] 是否避免在组件中定义组件？
- [ ] 是否导出了 Props 类型？

## Props
- [ ] Props 是否使用 interface 定义？
- [ ] 是否有合理的默认值？
- [ ] 是否避免直接修改 props？
- [ ] Props 命名是否清晰？

## 状态管理
- [ ] 状态是否放在最合适的位置？
- [ ] 是否避免不必要的 useState？
- [ ] 是否正确使用受控/非受控组件？
- [ ] 列表 key 是否唯一且稳定？

## 事件处理
- [ ] 事件处理函数是否以 handle 开头？
- [ ] 是否避免内联匿名函数？（不影响性能时允许）
- [ ] 表单事件是否阻止默认行为？
- [ ] 是否清理了事件监听？

## 条件/列表渲染
- [ ] 条件渲染是否使用三元或 &&？
- [ ] 是否避免 IIFE 条件渲染？
- [ ] 列表渲染是否提供了 key？
- [ ] 列表 key 是否不是 index？

## 性能
- [ ] 是否需要 React.memo？
- [ ] 是否避免了不必要的重渲染？
- [ ] 是否需要 useMemo/useCallback？
- [ ] 大型列表是否使用了虚拟化？

## 样式
- [ ] className 是否使用了 CSS Modules / Tailwind？
- [ ] 是否避免了内联样式？
- [ ] 是否有全局样式污染风险？
