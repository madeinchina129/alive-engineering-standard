# 用户增长策略 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| GR-001 | 所有增长策略必须有明确的假设和成功指标 | P0 | 是 |
| GR-002 | A/B 实验必须运行到统计显著（样本量充足，p < 0.05） | P0 | 是 |
| GR-003 | 获客成本（CAC）和用户生命周期价值（LTV）必须持续追踪 | P0 | 是 |
| GR-004 | 用户激活指标必须在新用户前 7 天内定义和追踪 | P0 | 是 |
| GR-005 | 增长实验必须记录实验文档（假设、设计、结果、结论） | P1 | 是 |

## 详细说明

### GR-001（P0）
所有增长策略必须有明确的假设和成功指标

### GR-002（P0）
A/B 实验必须运行到统计显著（样本量充足，p < 0.05）

### GR-003（P0）
获客成本（CAC）和用户生命周期价值（LTV）必须持续追踪

### GR-004（P0）
用户激活指标必须在新用户前 7 天内定义和追踪

### GR-005（P1）
增长实验必须记录实验文档（假设、设计、结果、结论）
