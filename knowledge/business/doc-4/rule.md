# 需求分析方法论 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| RA-001 | 需求分析必须包含：业务描述、用户场景、功能需求、非功能需求 | P0 | 是 |
| RA-002 | 需求必须使用用户故事（User Story）格式描述功能需求 | P0 | 是 |
| RA-003 | 每个用户故事必须有完整的 Acceptance Criteria | P0 | 是 |
| RA-004 | 非功能需求（性能/安全/可用性/可扩展性）必须明确度量标准 | P0 | 是 |
| RA-005 | 需求依赖关系必须绘制依赖图谱，识别关键路径 | P1 | 推荐 |
| RA-006 | 模糊需求必须通过原型或交互验证后再进入开发 | P1 | 是 |

## 详细说明

### RA-001（P0）
需求分析必须包含：业务描述、用户场景、功能需求、非功能需求

### RA-002（P0）
需求必须使用用户故事（User Story）格式描述功能需求

### RA-003（P0）
每个用户故事必须有完整的 Acceptance Criteria

### RA-004（P0）
非功能需求（性能/安全/可用性/可扩展性）必须明确度量标准

### RA-005（P1）
需求依赖关系必须绘制依赖图谱，识别关键路径

### RA-006（P1）
模糊需求必须通过原型或交互验证后再进入开发
