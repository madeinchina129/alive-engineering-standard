```markdown
# Sprint 流程指南

## Sprint 节奏（2 周）
| 天 | 活动 | 时长 | 参与人 |
|----|------|------|--------|
| 周一 | Sprint Planning | 2h | 全体 |
| 周一-周四 | 开发 | - | 开发团队 |
| 每天 | Daily Standup | 15min | 全体 |
| 周五下午 | Sprint Review | 1h | 全体+相关方 |
| 周五下午 | Retrospective | 1h | 全体 |

## Sprint Planning 议程
1. Product Owner 介绍 Sprint Goal (10min)
2. 团队评估 Backlog 条目 (30min)
3. 任务分解和认领 (30min)
4. 风险识别和应对 (20min)
5. 确认 DoD (10min)

## Definition of Done (DoD)
- [ ] 代码已合并到主分支
- [ ] 单元测试通过（覆盖率 ≥ 80%）
- [ ] 集成测试通过
- [ ] 代码评审通过
- [ ] 产品负责人验收
- [ ] 文档已更新

## 站会三问
1. 昨天我做了什么？
2. 今天我要做什么？
3. 有什么阻碍？
```
