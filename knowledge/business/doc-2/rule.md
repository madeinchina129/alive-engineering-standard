# 产品管理规范 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| PM-001 | 每个功能需求必须有明确的业务目标和成功度量指标 | P0 | 是 |
| PM-002 | 需求必须经过「价值评估 → 技术评审 → 排期」流程 | P0 | 是 |
| PM-003 | 需求文档须包含：用户故事、验收标准、边界条件、埋点方案 | P0 | 是 |
| PM-004 | 每个迭代必须有明确的 Scope 和 Done 标准 | P0 | 是 |
| PM-005 | 功能上线后 2 周内需产出效果复盘报告 | P1 | 推荐 |
| PM-006 | 需求变更需触发重新评估，不能随意插入迭代 | P1 | 是 |

## 详细说明

### PM-001（P0）
每个功能需求必须有明确的业务目标和成功度量指标

### PM-002（P0）
需求必须经过「价值评估 → 技术评审 → 排期」流程

### PM-003（P0）
需求文档须包含：用户故事、验收标准、边界条件、埋点方案

### PM-004（P0）
每个迭代必须有明确的 Scope 和 Done 标准

### PM-005（P1）
功能上线后 2 周内需产出效果复盘报告

### PM-006（P1）
需求变更需触发重新评估，不能随意插入迭代
