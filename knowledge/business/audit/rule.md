# 数据驱动决策 — 规则

## 规则列表

| 规则 | 说明 | 优先级 | 强制 |
|------|------|--------|------|
| DD-001 | 所有产品上线/改版必须设计 A/B 实验或效果评估方案 | P0 | 是 |
| DD-002 | 数据分析结论必须明确置信区间和统计显著性 | P0 | 是 |
| DD-003 | 业务报告必须区分「事实」和「观点」，标注数据来源 | P0 | 是 |
| DD-004 | 异常数据变化（超过 3σ）需在 24h 内发出告警 | P1 | 推荐 |
| DD-005 | 团队每月至少进行一次数据分享会 | P1 | 推荐 |

## 详细说明

### DD-001（P0）
所有产品上线/改版必须设计 A/B 实验或效果评估方案

### DD-002（P0）
数据分析结论必须明确置信区间和统计显著性

### DD-003（P0）
业务报告必须区分「事实」和「观点」，标注数据来源

### DD-004（P1）
异常数据变化（超过 3σ）需在 24h 内发出告警

### DD-005（P1）
团队每月至少进行一次数据分享会
